数组和字符串

A 集合 列表 数组

集合

定义: 集合一般被定义为- 一个或者多个确定的元素所构成的整体

特性:

  • 集合里面的元素类型不一定相同
  • 集合里面的元素没有顺序(但是编程语言中的数据结构都是封装添加了一些功能实现了顺序的存储等)

列表

定义: 按照一定(线性)顺序, 排列而成的数据项的集合.

特性: 列表的概念是在集合的特征上形成的, 有序, 长度可变.

数组

定义: 数组会用索引(数字)来标识每项数据在数组中的位置, 大多数索引都是从0开始,可以根据数组中的索引快速访问数组中的元素

特性: 数组是列表的实现方式之一(但是列表中没有索引). 数组的元素在内存中是连续存储的,并且每个元素占用相同大小的内存. 列表的元素则相反可能相邻可能不相邻

A 数组的操作

读取元素

计算机中的内存(形象):

创建数组的时候, 计算机会在内存中为数组申请一段连续 的空间, 然后0索引的位置则会被记录, 其他位置不会被记录

访问索引2 (元素 “D”) 时, 计算机的操作如下:

  • 找到数组索引为0 的内存地址 : 2008
  • 计算: 内存地址 + 想要搜索的索引值 = 目标元素地址 >>> 2008 + 2 = 2010 从而找到对应元素 D

时间复杂度 O (1)

查找元素

如果并不知道 索引所对应的元素 ,只是想找到数组中是否含有这个元素, 我们需要将整个数组遍历一遍才能得出这个元素书否存在

从0️⃣索引开始往后查找即可

最坏情况是查找完整个数组得知在最后一个元素或者没有此元素的时候, 如果数组长度是5我们则需要查找5次才, 数组长度为n则需要查找n次

时间复杂度 O (n)

图中为伪代码:

  1. 将_l r_ 的取值分别至于下标的外侧 -1 和 N (即当整个区域被_l 或者 r_ 覆盖他们相互不会处在对方的区域内)

  2. 既然要让_l r_ 永不交叉所以 当能进入循环体时r 的值为 1 即 r 的最小值

  3. 这样做之后将m(中间值)确定在了一个[0, N)的区间, 通过确定m的极值即可证明m被限制在了下标有效的边界

  4. 因为将整个数组下标分为了红蓝区域并且l r 都是从界外开始移动所以直接抛弃l r= m+-1 这种写法

  5. 所以在题目要求取得确定值的时候就不需要再将if分成三个部分, 只需要 判断 在m处替换能得到想要的结果的判断语句即可

所以在LC35这道题里面 if 条件里面给了谁闭区间直接在这个判断中嵌套if判断是否和目标值相等即可得到target的下标, 如果未寻得最后return right 即可得到插入的下标

LC56: 合并区间

  1. 判断二位数组是否在纵向为0. 为0是可以理解为二维数组的一维状态, 由于题目限制二维状态下只有两个元素,所以当纵向为0时直接return new int[0][2]

  2. 通过java.util包下的Arrays工具类找到sort通过比较器进行排序的方法, 方法的详情如下:

    切记Comparator 比较器不能传入非引用类型所以代码中传入的是二维数一维纵向的引用类型 int[] interval1, int[] interval2. 然后在通过引用指向自身的二维横向的第一个元素 interval1[0] - interval2[0]—>等价于 int [][0]这样就可以通过比较器对二维数组的每一个元素区间的—左区间指定升序排列规则(interval1[0] - interval2[0])降序反之.

  3. 通过一个ArrayList存储最终的答案

    • 排序过后的二维数组会以如下方式进行排列

    • 将区间按顺序加入ArrayList之中

      • 如果即将加入的区间的左端点在数组中最后一个区间的右端点之后, 他们不会重合, 我们可以直接将这个区间加入数组的末尾

        例: 当前数组最后一个区间为(0,9),即将加入的区间为 (10,11)—>即将加入的区间的左端点 10; 数组中最后一个区间的右端点 9. 此时 9<10 10位于数组内最后一个区间的右端点之后, 直接将 (10,11) 这个区间加在数组末尾即可.

      • 否则,他们将重合, 我们需要用当前区间的右端点 更新数组中最后一个区间的右端点, 将其置为二者的较大值

        例: (0,9)为当前数组最后一个区间, 即将添加 (2,5)进入数组, 由于 9 !< 2, 所以更新 (0,9)的右端点 9 , 将数组中最后一个区间的右端点(9) 和即将加入数组的右端点(5)进行比较选出最大值,并且将其赋值给数组中最后一个区间的右端点(进行合并)

  4. 最后转成数组并且返回return list.toArray(new int[list.size()][])

    将 new int[list.size()][] 视为int[]类型的数组[]也就是我们所说的二维数组

    注意 int[list.size()][]这样写的性能并没有 直接 int[0][] 性能好, 这类型的java数组并不需要进行空间的初始化 int[0]就像 String 一样是一个引用类型.
